    第五讲 文本编辑器Vim

    5.1 文本编辑器Vim-Vim常用操作
插入命令： a: 在光标的后面插入。
A: 在光标的行尾插入。
i: 在光标前插入。
I:在光标的行首插入。
o:在光标下插入。
O:在光标上插入新行。
删除命令： x:删除光标所在处字符。
nx:删除光标后n个字符。
dd:删除光标所在行。
ndd:删除光标后n行。
dG:从光标所在行删到文件末尾。
D:删除光标所在处至行尾。
:n1,n2d:删除指定范围的行。
编辑命令： :set nu:设置行号。
:set nonu:取消行号。
gg:到首行。
GG:到尾行。
nG:到第n行。
:n:到第n行。
$:到行首:。
0:到行尾。
复制、剪切命令： yy:复制当前行。
nyy:复制当前及以下n行。
dd:剪切当前行。
ndd:剪切当前行以及以下n行。
p:粘贴在光标行下。
P:粘贴在光标行上。
其他命令： r:取代光标处字符。
R:从光标处开始替换字符，Esc结束。
u:取消上一步操作。
查找、替换 /你要搜索的字符串:查找区分大小写。
n:下一个结果。
:set ic:不区分大小写。
:%s/old_string/new_string/g:全文替换。
:n1,n2s/old_string/new_string/g:一定范围内的替换 ，g替换不询问,c替换之前询问。
保存、退出 :w:保存修改。
:w new_filename:另存为指定文件。
:wq:保存修改并退出。
ZZ:保存修改并退出。
:q!:不保存而退出。
:wq!:保存修改并退出（文件所有者和root使用）。

6.1 软件包管理-简介

软件包：源码包（脚本安装包）、二进制包（RPM包、系统默认包）。

6.2 RPM- rpm命令管理：

  • 挂载光盘：1.导入光盘。2.创立挂载点mkdir /mnt/cdrom。3.将光盘导入挂载点mount /dev/sr0 /mnt/cdrom。
  • 安装软件：rpm -ivh 软件包全名。-i(install) -v(显示详细信息) -h(安装进度) --nodeps(不检测依赖性、不要使用，哈哈)
  • 升级软件：rpm -Uvh 软件包全名，-U(upgrade升级)。
  • 软件卸载：rpm -e 包名，-e卸载， --nodeps不依赖。
  • 对于有依赖性的软件（如果是库的话通过www.rpmfind.net查询）。
  • 安装和升级使用包全名。其他情况使用包名，在var/lib/rpm下。不一定要在/Packages目录下。

查询：

rpm -q 包名：-q query查询是否安装。
rpm -qa：查询所有安装的包。
rpm -qa | grep 关键字：查询所有有关键词的软件。
rpm -qi 包名：查询软件包详细信息。
rpm -ql 包名：查询软件包安装位置。
rpm -qlp 包全名：在Packages下查询未安装包将要安装的位置。
rpm -qf 系统文件名：查询是属于哪个软件。系统文件名 文件等绝对路径。
rpm -qR 包名：查询已安装包的依赖性。
rpm -qp 包全名：查询未安装包的依赖性。

校验、文件提取。

rpm -V 已安装的包名：-V verify验证。
rpm2cpio /mnt.cdrom/Packages/xxxx.rpm | cpio -idv ./bin/ls 把RPM包内的ls命令提取到当前目录到/bin/ls下。

6.3 RPM- yum在线管理：解决软件依赖性。

IP地址配置、

  • setup 设置各种、save.
  • vim /etc/sysconfig/network-scripts/ifcfg-eth0 把ONBOOT="no"改完yes。
  • service network restart 重启网络。

网络yum源：

  • cd /etc/yum.repos.d/
  • 能联网，使用CentOS-Base.repo
  • 不能联网，使用CentOS-Media.repo，让本地光盘成为yum源。

yum常用命令：牛逼的yum将要依赖的软件自动下载安装了。

yum list:查询可用软件包列表。
yum search 关键字： 查询有关键字的软件包。
yum -y install 包名：安装软件包（-y 是否安装提示。软件包例如gcc）
yum -y update 包名：升级。
yum -y update：如果没有包名就完蛋了，全部软件甚至是内核都会升级。
yum -y remove 包名：不仅仅卸载你要的软件，把依赖的包都删掉了，傻逼了。
yum grouplist:列出所有可用的软件组列表。一下子安装软件包组 比 单独安装方便。
yum groupinstall "软件组名"
yum groupremove "软件组名"

光盘作为本地yum

10.3.3 Shell基础-Bash基本功能-输入、输出重定向

设备 设备文件名 文件描述符 类型
键盘 /dev/stdin 0 标准输入
显示器 /dev/sdout 1 标准输出
显示器 /dev/sdterr 2 标准错误输出
  • 命令 > 文件：以覆盖的形式，把正确的结果写入文件。
  • 命令 >> 文件：以追加的形式，把正确的结果写入文件。
  • 错误命令 2> 文件：以覆盖的形式，把错误结果写入文件。
  • 错误命令 2>> 文件：以追加的形式，把错误结果写入文件。
  • 命令（不论正确还是错误）>> 文件 2>&1，命令（不论正确还是错误）&>> 文件:不论错误还是正确，均将结果追加到文件里面。
  • 命令（不论正确还是错误）> 文件 2>&1，命令（不论正确还是错误）&> 文件:不论错误还是正确，均将结果覆盖到文件里面。
  • 命令>>文件1 2>>文件2：正确的结果追加到文件1，错误的结果追加到2，2>>文件2中间没有空格。

10.3.4 Shell基础-Bash基本功能-多命令顺序执行与管道符。``

多命令执行符 格式 作用
; 命令1;命令2 依次执行，么有关系
&& 命令1&&命令2 只有当命令1执行成功，命令2才会执行
\ \ 命令1\ \ 命令2 只有当命令1执行不成功，命令2才会执行
  • 管道符:命令1|命令2,命令1的正确输出结果作为命令2的操作对象。netstat -an | grep established

10.3.5 Shell基础-Bash基本功能-通配符和其他特殊符号

通配符 作用
? 匹配任意一个字符,ls ?sh匹配文件名为3个，后面为sh。
* 匹配0或者多个任意字符
[ ] 匹配括号中任意一个字符
[a-z] 匹配一个范围内的任意一个字符
[\^0-9] 匹配任意一个不是数字的字符。
